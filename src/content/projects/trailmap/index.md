Trailmap is an offline-first map concept for planning routes and checking recent trail conditions. The project focuses on resilience: the core experience should still work when service is unreliable.

The interesting design challenge was deciding what information matters when someone is already outside. I kept the interface sparse, prioritizing route state, last-known condition reports, and notes that can be saved quickly.
